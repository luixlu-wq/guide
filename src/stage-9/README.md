# Stage 9 Runnable Runbook (Strict Execution)

This folder is the operatable execution guide for Stage 9 (AI system architecture).
Use it as an evidence workflow, not only a reading reference.

---

## 1) Setup (Required)

Run inside `red-book/src/stage-9`:

```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

Optional richer path (CUDA and architecture ecosystem packages):

```powershell
pip install -r requirements-optional.txt
```

---

## 2) Preflight Checks

```powershell
python --version
python -c "import numpy; print('numpy ready')"
python -c "import torch; print('torch', torch.__version__, 'cuda_available', torch.cuda.is_available())"
```

If torch is not installed, PyTorch/CUDA scripts still run with deterministic CPU fallback.

---

## 3) Run Modes

Fail-fast core path:

```powershell
powershell -ExecutionPolicy Bypass -File .\run_all_stage9.ps1
```

Full ladder path (`simple -> intermediate -> advanced`):

```powershell
powershell -ExecutionPolicy Bypass -File .\run_ladder_stage9.ps1
```

Ladder + labs:

```powershell
powershell -ExecutionPolicy Bypass -File .\run_ladder_stage9.ps1 -IncludeLabs
```

## 8) Troubleshooting Quick Guide

If scripts fail on imports:

1. activate venv
2. rerun `pip install -r requirements.txt`
3. verify `python` path with `Get-Command python`

If CUDA topic scripts use fallback:

1. check `python -c "import torch; print(torch.cuda.is_available())"`
2. verify GPU runtime and driver
3. keep CPU fallback path for deterministic baseline

If Qdrant track is unavailable:

1. start local Qdrant on `localhost:6333`
2. rerun topic/lab scripts
3. check status column in generated metrics
